# Project Aurora - Documentation

This directory contains all project documentation for Project Aurora, organized by purpose and audience.

## üìÇ Directory Structure

### `planning/`
**Strategic planning, decision logs, and development history**

- **PLANNING.md** - Structured planning framework tracking progress through 5 development phases
- **BACKLOG.md** - Feature stories and acceptance criteria organized by milestone
- **PROJECT_JOURNAL.md** - Chronological log of development sessions, discussions, and decisions

**Purpose:** Captures the evolution of project scope, rationale behind key decisions, alternatives considered, and progress tracking. Ensures continuity across development sessions.

**Use when:**
- Reviewing milestone progress or strategic direction
- Tracking active stories during development
- Revisiting past decisions or understanding why certain approaches were chosen
- Onboarding collaborators or planning next steps

---

### `technical/`
**Architecture, design patterns, and troubleshooting**

- **ARCHITECTURE.md** - System structure, component map, design patterns, and data flow
- **DEBUGGING.md** - Recurring technical issues, platform-specific solutions, and troubleshooting guides

**Purpose:** Documents the technical foundation of the application, including architectural decisions and operational knowledge for resolving common issues.

**Use when:**
- Understanding the system's structure or component interactions
- Investigating build issues, deployment problems, or platform-specific bugs
- Making architectural decisions or evaluating refactoring approaches

---

### `guides/`
**Developer workflow and contribution standards**

- **CONTRIBUTING.md** - Developer onboarding, local setup, branching workflow, testing requirements, and coding standards

**Purpose:** Provides actionable guidance for contributors to set up their environment, follow project conventions, and maintain code quality.

**Use when:**
- Setting up the development environment for the first time
- Submitting pull requests or creating feature branches
- Verifying coding standards or running tests locally

---

### `design/`
**Visual design system and assets**

- **DESIGN_SYSTEM.md** - Visual language, color palette ("Morning Mist"), typography, spacing, and UI patterns
- **visual/** - Design assets and interactive mockups

**Purpose:** Defines the aesthetic foundation of the application, ensuring visual consistency and accessibility across all UI components.

**Use when:**
- Implementing new UI features or components
- Reviewing design decisions for colors, fonts, or spacing
- Creating marketing materials or documentation screenshots
