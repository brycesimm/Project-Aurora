# .codex Directory

This directory contains project documentation, decision logs, and development notes for Project Aurora.

## Contents

- **PLANNING.md** - Structured planning framework tracking progress through 5 development phases
- **PROJECT_JOURNAL.md** - Chronological log of development sessions, discussions, and decisions
- **README.md** - This file; explains the purpose of the .codex directory

## Purpose

The .codex directory serves as a knowledge base for the project, capturing:
- Rationale behind key decisions
- Alternatives considered and why they were rejected
- Evolution of project scope and features
- Technical research and findings
- Development challenges and solutions

This documentation ensures continuity across development sessions and provides context for future decisions.

## Usage

Update PROJECT_JOURNAL.md after significant discussions or decisions. Use it as a reference when:
- Revisiting past decisions
- Onboarding collaborators
- Remembering why certain approaches were chosen
- Planning next steps based on previous discussions
